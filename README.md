# Inbet — Кариерна страница

Самостоятелна HTML страница с отворени позиции, която се синхронизира автоматично с Excel файл (`data.xlsx`). Всеки ред от таблицата става отделна сгъваема позиция на сайта.

## Какво съдържа

- **`index.html`** — цялата страница (дизайн, логика, лого и снимки), един-единствен файл.
- **`data.xlsx`** — таблицата с позициите, която ти редактираш.

> Логото и всички снимки от слайдшоуто на сградата/офиса са **вградени** в `index.html` (като base64), затова **не са нужни отделни файлове за изображения**. В репото качваш само тези два файла.

## Структура на репото

Двата файла трябва да са в **една и съща папка** (най-добре в основата на репото):

```
твоето-репо/
├─ index.html     ← страницата
└─ data.xlsx      ← таблицата с позициите
```

## Качване в GitHub Pages (през браузъра, без команди)

1. Влез в GitHub → **New repository**. Дай му име (напр. `inbet-careers`) и го направи **Public** (безплатният GitHub Pages работи с публични репота).
2. В новото репо: **Add file → Upload files**. Плъзни **и двата файла** — `index.html` и `data.xlsx` — и натисни **Commit changes**.
3. Отвори **Settings → Pages**.
4. Под **Source** избери **Deploy from a branch**, после branch **`main`**, папка **`/ (root)`**, и натисни **Save**.
5. Изчакай около минута. Сайтът е готов на адрес:
   `https://<твоето-потребителско-име>.github.io/<име-на-репото>/`

Тъй като файлът се казва `index.html`, сайтът се отваря директно от основния адрес.

## Как да добавяш / махаш позиции

1. Отвори `data.xlsx` локално и редактирай: **нов ред = нова позиция**, **изтрит ред = махната позиция**.
2. В репото: **Add file → Upload files** → качи новия `data.xlsx` (върху стария) → **Commit changes**.
3. Презареди страницата. Ако още виждаш старите данни, направи **твърдо презареждане**: `Ctrl + F5` (Windows) или `Cmd + Shift + R` (Mac).

Броят сгъваеми менюта и филтрите по отдел се настройват автоматично спрямо съдържанието на таблицата — няма ограничение за броя позиции.

## Колони в `data.xlsx` (имената трябва да съвпадат точно)

| Колона | Какво става с нея на страницата |
|---|---|
| **Position** | Заглавие на позицията (името на сгъваемото меню) |
| **Department** | Блок „Отдел" над „Заплата"; захранва и филтрите по отдел |
| **Position Summary** | Въвеждащ параграф най-горе в позицията |
| **Key Responsibilities** | Точки под „Какво ще правиш" (всеки нов ред в клетката = нова точка) |
| **Requirements** | Точки под „Какво ще донесеш" (всеки нов ред в клетката = нова точка) |
| **Salary range min** | Начало на диапазона в „Заплата" |
| **Salary range max** | Край на диапазона в „Заплата" |

> Ако клетките за заплата са празни (или не са числа), редът „Заплата" просто не се показва за тази позиция.

## Важно

- Сайтът трябва да се отваря през **уеб адрес (GitHub Pages)**, а не директно като файл (`file://`). При двоен клик върху `index.html` браузърът блокира четенето на локалния `data.xlsx`, затова позициите няма да се заредят (виж следващата точка).
- Името на файла трябва да е точно **`data.xlsx`** (с малки букви) и да стои до `index.html`.
- При проблем със зареждането страницата показва съобщение какво да се провери.
- За да смениш снимките в слайдшоуто или логото, трябва да се прегенерира `index.html` (те са вградени във файла, не са отделни файлове).
